변경사항 

초기 프로젝트파일에서는 엔터티 생성과 세션기반 로그인만 구현했으나 JWT를 사용한 로그인방식으로 변경함 
기존에는 세션기반으로 로그인 상태유지를 해왔으나 현재는 매 페이지마다 Common.js의 CheckLogin api 가 호출됨 이 함수에서는 로컬스토리지에 액세스토큰을 불러와 헤더에 authorization 정보를 추가해줌 그다음 해당 api를 위한 컨트롤러에서는 jwtUtil이라는 컴포넌트 클래스 파일을 만들어 검증,생성,리프레시토큰,id추출과 같은 메소드들이 들어가있기에 헤더정보에서 토큰값을 추출하여 로그인 상태가 유효한지 체크하고있음 지식의 한계인지 본문 자체에 header를 추가하는게 안되고있음.
